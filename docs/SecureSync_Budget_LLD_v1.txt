1. INTRODUCTION
================================================================================

SecureSync Budget is a personal budget management Android application built
with Jetpack Compose and Material Design 3. The application enables users to
track income and expenses, configure recurring financial entries, amortize
large purchases over time, set and manage savings goals, and visualize
spending through interactive charts. A distinctive Solari-style split-flap
display animates the user's available cash on the dashboard. The app supports
two languages (English and Spanish), multiple currency formats, encrypted
file save/load functionality, and CSV import from US Bank statements. All
domain data is persisted as JSON files in the application's private storage
directory, with user preferences stored in SharedPreferences. The architecture
follows a single-activity Compose pattern where MainActivity serves as the
central state coordinator, delegating UI rendering to composable screen
functions and data persistence to singleton repository objects.

This Low-Level Design document describes every class, composable function,
data model, utility object, and persistence schema in the SecureSync Budget
application. Each entry includes its package, purpose, constants, fields,
methods with detailed descriptions, and relevant design notes.

Methods:

    onCreate(savedInstanceState: Bundle?)
        Entry point for the activity. Calls enableEdgeToEdge() for full-screen
        rendering, then enters a setContent block where all Compose state is
        declared, data is loaded from repositories, auto-provisioned categories
        are ensured, the budget period refresh logic runs, and the screen
        router dispatches to the correct composable based on currentScreen.

    saveIncomeSources()
        Delegates persistence of the current incomeSources list to
        IncomeSourceRepository.save(). This function is called after any
        mutation to the income sources collection to ensure disk consistency.

    saveRecurringExpenses()
        Delegates persistence of the current recurringExpenses list to
        RecurringExpenseRepository.save(). Called after additions, edits,
        or deletions of recurring expense entries.

    saveAmortizationEntries()
        Delegates persistence of the current amortizationEntries list to
        AmortizationRepository.save(). Called after any mutation to the
        amortization entries collection.

    saveSavingsGoals()
        Delegates persistence of the current savingsGoals list to
        SavingsGoalRepository.save(). Called after additions, edits,
        deletions, supercharge operations, or period refresh updates.

    saveTransactions()
        Delegates persistence of the current transactions list to
        TransactionRepository.save(). Called after any mutation to the
        transaction collection including imports.

    saveCategories()
        Delegates persistence of the current categories list to
        CategoryRepository.save(). Called after additions, edits, or
        deletions of category entries.

    persistAvailableCash()
        Writes the current availableCash value to SharedPreferences as a
        float. This is called after every operation that modifies the
        available cash balance to ensure the value survives app restarts.

    recalculateBudget()
        Invokes BudgetCalculator.calculateSafeBudgetAmount() with the current
        income sources, recurring expenses, and budget period, then stores the
        result in both safeBudgetAmount state and SharedPreferences. This is
        called when income sources or recurring expenses are modified.

    isBudgetAccountedExpense(txn: Transaction): Boolean
        Returns true if the given expense transaction has category amounts
        assigned to the "Recurring" or "Amortization" categories. These
        transactions are already factored into the safe budget calculation and
        should not be double-counted when adjusting availableCash.

    addTransactionWithBudgetEffect(txn: Transaction)
        Adds a transaction to the transactions list, saves to disk, and
        adjusts availableCash accordingly. For non-budget-accounted expenses
        that fall on or after the budget start date, the amount is subtracted
        from availableCash; for non-budget income, the amount is added.

    runMatchingChain(txn: Transaction)
        Executes the sequential matching chain for a newly entered transaction.
        The chain tests in order: duplicate detection, recurring expense match,
        amortization match, and budget income match. If any match is found, the
        corresponding confirmation dialog state is set. If no match is found,
        the transaction is saved directly via addTransactionWithBudgetEffect().

    Budget Period Refresh Logic (inline in remember block):
        Executes once on app launch. Calculates the number of missed budget
        periods between lastRefreshDate and today using
        BudgetCalculator.countPeriodsCompleted(). For each missed period, adds
        the current budgetAmount to availableCash and updates each non-paused
        savings goal's totalSavedSoFar by its per-period deduction amount.
        Persists all updated state to SharedPreferences and repositories.

    Supercharge Handler (onSupercharge lambda):
        Processes savings goal supercharge allocations. For each allocation,
        caps the amount at the remaining goal balance, then applies the
        appropriate SuperchargeMode. For target-date goals with ACHIEVE_SOONER,
        recalculates and moves the target date earlier based on the current
        contribution rate and the new remaining balance. For fixed-contribution
        goals with REDUCE_CONTRIBUTIONS, lowers contributionPerPeriod
        proportionally to maintain the same payoff timeline. For all other
        cases, simply adds the lump sum to totalSavedSoFar. Deducts the total
        allocated amount from availableCash.

Import Pipeline:
    The import pipeline is implemented as a LaunchedEffect state machine.
    It transitions through ImportStage values: FORMAT_SELECTION allows the user
    to choose a bank format; PARSING reads the file using the appropriate parser;
    PARSE_ERROR displays errors; DUPLICATE_CHECK runs filterAlreadyLoadedDays()
    to pre-filter and then processes each transaction through the matching chain;
    COMPLETE adds surviving transactions to the main list.

    - Auto-matching chain during import for recurring, amortization, and income

    formatCurrency(amount: Double, currencySymbol: String): String
        Formats a numeric amount with the correct number of decimal places for
        the given currency symbol. Looks up decimal count from CURRENCY_DECIMALS
        (defaulting to 2) and returns a string with the currency symbol prefix
        and formatted number.

    buildSignCurrencyValues(currencies: List<String>): List<String>
        Builds the ordered list of values that the sign/currency flip card
        cycles through. For each currency symbol, adds the positive form and
        the negative form (prefixed with "-"). Multi-character symbols like
        "Fr" are split across two lines with "-" on top.

    pow10(n: Int): Int (private)
        Computes 10 raised to the power n using iterative multiplication.
        Used to split an integer amount into whole and decimal parts based
        on the number of decimal places.

Composable Functions:

    FlipDisplay(amount, isNegative, currencySymbol, digitCount, decimalPlaces, soundPlayer, modifier, bottomLabel)
        Main display composable that renders the complete Solari-style flip
        board. Uses BoxWithConstraints to compute responsive card widths from
        available space, clamped to MAX_CARD_WIDTH. Splits the integer amount
        into whole and decimal digit arrays, replaces leading zeros with blank
        (-1) values, and renders a FlipChar for the sign/currency card followed
        by FlipDigit composables for each digit position. An optional
        DecimalDot separates whole and decimal digits. Supports an optional
        bottomLabel text below the card row.

    DecimalDot(cardHeight, fontSize, color) (private)
        Renders the decimal point between whole and decimal digit cards.
        Uses a fixed-width Box aligned to BottomCenter with a bold monospace
        period character, sized to match the surrounding card proportions.

File:       CryptoHelper.kt
Package:    com.syncbudget.app.data

10. PERSISTENCE SCHEMA
================================================================================


All domain data is persisted as JSON files in the application's private
internal storage directory (Context.openFileOutput with MODE_PRIVATE).
User preferences are stored in SharedPreferences under the name "app_prefs".


--------------------------------------------------------------------------------
10.1  transactions.json
--------------------------------------------------------------------------------

Repository:     TransactionRepository
File Name:      transactions.json
Format:         JSON Array of objects

JSON Schema:
Field	Type	Required	Description
id	Integer	Yes	Unique transaction identifier (0-65535)
type	String	Yes	TransactionType enum name ("EXPENSE" or "INCOME")
date	String	Yes	ISO-8601 date string (e.g. "2026-02-20")
source	String	Yes	Merchant or income source name
amount	Double	Yes	Transaction amount (always positive)
isUserCategorized	Boolean	No*	Whether user manually set categories (default true)
isBudgetIncome	Boolean	No*	Whether matched to a budget income source (default false)
categoryAmounts	Array	No	Array of {categoryId: Int, amount: Double} objects

*Fields default to true/false respectively when absent during load.

Example:
    {
        "id": 42301,
        "type": "EXPENSE",
        "date": "2026-02-15",
        "source": "Grocery Store",
        "amount": 45.67,
        "isUserCategorized": true,
        "isBudgetIncome": false,
        "categoryAmounts": [
            {"categoryId": 1001, "amount": 30.00},
            {"categoryId": 1002, "amount": 15.67}
        ]
    }

All five repositories follow an identical pattern: Kotlin object singletons
with a private FILE_NAME constant, a save() method that serializes to
JSONArray and writes via Context.openFileOutput, and a load() method that
reads via Context.openFileInput, parses the JSONArray, and returns a list
of domain objects.

Package:    com.syncbudget.app.data
File Name:  transactions.json

Methods:

    save(context: Context, transactions: List<Transaction>)
        Serializes the transaction list to a JSONArray and writes to
        transactions.json in the app's private storage. Each transaction is
        converted to a JSONObject with all scalar fields. The categoryAmounts
        list is serialized as a nested JSONArray only when non-empty, with each

12. ERROR HANDLING
================================================================================


The application employs the following error handling strategies:

12.1  CSV Import Errors
    - parseUsBank() and parseSyncBudgetCsv() catch exceptions per line and
      accumulate error messages with line numbers in the CsvParseResult.error
      field. The UI displays a parse error dialog when errors are present.
    - parseSyncBudgetCsv() validates the CSV header and returns an immediate
      error if it does not match the expected format.

12.2  Encryption/Decryption Errors
    - CryptoHelper.decrypt() throws IllegalArgumentException if the input data
      is too small to contain salt + nonce + ciphertext. The UI catches this
      and displays an appropriate error message.
    - Invalid passwords result in javax.crypto.AEADBadTagException during
      Poly1305 tag verification, caught by the import pipeline.

12.3  Preference Loading Errors
    - BudgetPeriod.valueOf() is wrapped in a try-catch that defaults to
      BudgetPeriod.DAILY if the stored preference string is invalid.
    - All SharedPreferences reads use explicit default values.

12.4  Repository Loading Errors
    - All repository load() methods check for file existence and blank content,
      returning empty lists rather than throwing exceptions.
    - SavingsGoalRepository.load() wraps targetDate parsing in a try-catch,
      defaulting to null on parse failure.

12.5  ID Collision Handling
    - All ID generation functions use do-while loops to retry random generation
      until a unique value is found within the 0..65535 range.
    - parseSyncBudgetCsv() checks imported IDs against existingIds and
      generates new IDs on collision.

12.6  Budget Calculation Safety
    - BudgetCalculator.calculateSafeBudgetAmount() returns 0.0 if
      periodsPerYear is zero, preventing division by zero.
    - countPeriodsCompleted() returns 0 if the end date is not after the start
      date.
    - activeSavingsGoalDeductions() skips goals where remaining balance is
      zero or negative, and where remaining periods is zero.

12.7  UI Input Validation
    - Amount input fields use currency-aware decimal limiting to prevent
      invalid numeric entries.
    - RepeatType-specific form fields dynamically appear/disappear to prevent
      submission of incomplete recurrence configurations.
    - Protected categories ("Other", "Recurring", "Amortization",
      "Recurring Income") cannot be deleted through the UI.

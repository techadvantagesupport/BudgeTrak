1. Introduction

BudgeTrak is a personal budget management Android application built with Jetpack Compose and Material Design 3. The application enables users to track income and expenses, configure recurring financial entries, amortize large purchases over time, set and manage savings goals, simulate cash-flow projections, capture receipt photos, and visualize spending through interactive charts. A distinctive Solari-style split-flap display animates the user's available cash on the dashboard.

The app supports two languages (English and Spanish), multiple currency formats, encrypted file save/load functionality, CSV import from US Bank statements, a home screen widget with quick-add transactions, and multi-device family synchronization via Firestore-native per-document encrypted sync. All domain data is persisted as JSON files in the application's private storage directory, with user preferences stored in SharedPreferences.

The architecture follows a single-activity Compose pattern where MainActivity serves as the central state coordinator, delegating UI rendering to composable screen functions and data persistence to singleton repository objects.

v2.2 Architecture Change: Firestore-Native Sync

In v2.2, the hand-rolled CRDT sync system (~4,000 lines: LamportClock, DeltaBuilder, DeltaSerializer, CrdtMerge, SnapshotManager, SyncEngine, IntegrityChecker, PeriodLedgerCorrector, SyncFileLock) was replaced with Firestore-native document storage (~1,860 lines). Each record is a separate Firestore document with per-field encryption. Firestore handles transport, offline queue, retry, and ordering. Data classes no longer carry per-field _clock values; only deviceId and deleted remain as sync fields. Real-time snapshot listeners replace the old periodic pull/push cycle. Receipt photo sync via Cloud Storage was added.

v2.3 Architecture Change: Shared Services & Background Period Refresh

In v2.3, sync merge logic and period refresh logic were extracted from MainActivity into shared services (SyncMergeProcessor, PeriodRefreshService) usable by both the foreground activity and a new PeriodRefreshWorker. The WidgetRefreshWorker was removed and replaced by PeriodRefreshWorker, which performs a short-lived Firestore sync, period refresh, cash recompute, and Firestore push — all from the background. Net result: ~220 lines moved out of MainActivity into 3 new files totaling 984 lines, and 1 file (105 lines) removed.

This Low-Level Design document describes every class, composable function, data model, utility object, and persistence schema in the BudgeTrak application. Each entry includes its file path, line count, purpose, key methods, and relevant design notes.

Key Methods

onCreate(savedInstanceState: Bundle?)
Entry point. Installs crash logger to Download/crash_log.txt, calls enableEdgeToEdge(), signs in anonymously to Firebase, enters setContent block where all state is declared, data loaded, categories provisioned, budget refresh runs, Firestore listeners attached via DisposableEffect, and screen router dispatches to composables.
saveTransactions() / saveCategories() / saveIncomeSources() / etc.
Persists data to the corresponding repository. No changed parameter — all records in the list are pushed via SyncWriteHelper. Each save function calls the repository's save() then iterates the list calling the corresponding SyncWriteHelper.pushXxx() for every record.
recomputeCash()
Deterministically recomputes availableCash from budgetStartDate, periodLedger, transactions, and recurringExpenses via BudgetCalculator.recomputeAvailableCash(). Called after sync merges and transaction mutations.
onBatchChanged (DisposableEffect callback)
Registered on the FirestoreDocSync instance. Receives List<DataChangeEvent> from real-time Firestore listeners. Delegates to SyncMergeProcessor.processBatch() which returns a MergeResult. Applies non-null collection lists from MergeResult to Compose state. Handles side effects: pushes conflicted transactions back to Firestore, deletes remapped categories from Firestore. Applies settings prefs. Saves to repositories and recomputes cash.
Deferred Period Refresh
Period refresh is deferred by ~200ms after Firestore listener initial snapshot to allow incoming period ledger entries to arrive. Solo users (not in sync group) skip the wait. Calls PeriodRefreshService.refreshIfNeeded() which handles ledger creation, SG accrual, RE set-aside, cash recompute, and SharedPrefs update. Updates in-memory state from RefreshResult, pushes changed records (ledger via createDocIfAbsent(), SG and RE via updateFields()) via SyncWriteHelper, and updates lastSaved* caches + lastSyncActivity.
Integrity Check
Every 5 minutes, compares local record counts against Firestore directly. Count mismatch triggers listener reattach. Cash mismatch triggers recomputeCash(). Publishes lastSeen to device metadata.
runMatchingChain(txn: Transaction)
Sequential matching pipeline for incoming transactions (manual or CSV import):
  1. Duplicate check: Calls findDuplicate() with matchDays, matchPercent, matchDollar, matchChars tolerances. If found, shows DuplicateResolutionDialog.
  2. Recurring expense match: Calls findRecurringExpenseMatch(). If matched, shows RecurringExpenseConfirmDialog. On confirm, sets linkedRecurringExpenseId and stores linkedRecurringExpenseAmount (the RE amount at link time).
  3. Amortization match: Calls findAmortizationMatch(). If matched, shows AmortizationConfirmDialog. On confirm, sets linkedAmortizationEntryId.
  4. Budget income match: Calls findBudgetIncomeMatch(). If matched, shows BudgetIncomeConfirmDialog. On confirm, sets isBudgetIncome=true, linkedIncomeSourceId, and linkedIncomeSourceAmount.
Auto-Provisioned Categories
On startup, checks for required system categories by tag. Missing categories are auto-created with localized names.

Budget Period Refresh Logic
On app launch, delegates to PeriodRefreshService.refreshIfNeeded() which:
  1. Loads all data from disk, computes budgetAmount via BudgetCalculator.computeFullBudgetAmount().
  2. Detects missed periods between lastRefreshDate and now.
  3. For each missed period: creates PeriodLedgerEntry (dedup check), accrues SG totalSavedSoFar and RE setAsideSoFar.
  4. Saves to disk, recomputes cash, updates SharedPrefs.
  5. Returns RefreshResult with only changed records (for Firestore push by caller).
When in a sync group, period refresh is deferred by ~200ms after initial Firestore listener snapshot to allow incoming period ledger entries from other devices to arrive first. Solo users skip the wait. PeriodRefreshService returns null if no periods are missed. The same service is called by PeriodRefreshWorker for background refresh.
Save Functions Pattern (v2.2+)
All save functions follow the same pattern:
  1. Call the repository's save(context, data) to persist to local JSON.
  2. If SyncWriteHelper is initialized (device in sync group), push all records via the corresponding SyncWriteHelper.pushXxx() method. SyncWriteHelper dispatches each push as fire-and-forget on IO thread; the Firestore SDK handles offline queue and retry.
No changed parameter is needed — FirestoreDocSync's diff-based pushRecord() compares against lastKnownState and only sends changed enc_* fields.
Tombstone Cleanup
On app startup, the admin device cleans up old tombstones. Checks the oldest device lastSeen timestamp, then deletes Firestore documents where deleted=true and the doc is older than that timestamp minus a 1-day buffer. Local JSON is also purged of tombstones on first launch after group join. Migration pushes skip tombstones entirely.
Crash Logger
Installs a custom UncaughtExceptionHandler that writes stack traces with device info (Android version, manufacturer, model) to Download/BudgeTrak/support/crash_log.txt before delegating to the default handler. Appends to file so crash history accumulates.
Firebase Anonymous Auth
Signs in anonymously on app start via FirebaseAuth.signInAnonymously(). Required for Firestore security rules (request.auth != null). Completely invisible to user. App continues even if auth fails — Firestore rejects requests until auth succeeds.
Background Worker Scheduling
Schedules PeriodRefreshWorker.schedule(context) alongside SyncWorker.schedule(context) during sync setup. PeriodRefreshWorker handles background period refresh, cash recompute, and widget update when the app is not in the foreground.

File: data/sync/PeriodRefreshWorker.kt (446 lines)  |  Package: com.syncbudget.app.data.sync  |  Extends: CoroutineWorker

15-minute periodic background worker that performs a short-lived Firestore sync, processes incoming changes via SyncMergeProcessor, runs period refresh via PeriodRefreshService, pushes results back to Firestore, and updates the home screen widget. Replaces WidgetRefreshWorker from v2.2.

doWork() Flow

  1. Foreground check: Reads MainActivity.isAppActive volatile flag; returns Result.success() immediately if the app is in the foreground (foreground handles its own sync and refresh).
  2. Firebase auth: Signs in anonymously (same pattern as SyncWorker).
  3. Firestore sync (if configured): Creates a temporary FirestoreDocSync instance, opens short-lived listeners with ~60-second timeout via CompletableDeferred, accumulates DataChangeEvents, then calls stopListeners().
  4. Process events: Calls SyncMergeProcessor.processBatch(). Saves merged data to disk via repositories. Applies settings to SharedPreferences.
  5. Period refresh: Builds RefreshConfig from SharedPreferences, calls PeriodRefreshService.refreshIfNeeded().
  6. Cash recompute fallback: If sync delivered changes but no period boundary was crossed, calls recomputeCashFromDisk() (private method that loads all data from disk and calls BudgetCalculator.recomputeAvailableCash()) to pick up any sync-delivered transaction changes.
  7. Push to Firestore: Pushes new period ledger entries via createDocIfAbsent(), updated SG via updateFields() with fieldUpdate(sg, setOf("totalSavedSoFar")), updated RE via updateFields() with fieldUpdate(re, setOf("setAsideSoFar", "isAccelerated")).
  8. Push sync side effects: Deletes remapped categories from Firestore, pushes back conflicted transactions.
  9. Update widget + device metadata: Calls BudgetWidgetProvider.updateAllWidgets() and updates device metadata via FirestoreService.updateDeviceMetadata() with lastSeen plus a cash fingerprint: a JSONObject containing {"cash": currentCash} passed as the fingerprintJson parameter, enabling the integrity check to detect cash mismatches across devices.

Private Methods

recomputeCashFromDisk()
Loads all data from disk (transactions, recurring expenses, income sources, savings goals, amortization entries, period ledger) via repositories, builds the budget start date and period from SharedPreferences, and calls BudgetCalculator.recomputeAvailableCash(). Saves the recomputed cash to SharedPreferences. Called when sync delivered changes but no period boundary was crossed, ensuring the widget and device metadata reflect the latest synced state.

14. Error Handling

14.1 CSV Import Errors

Line-level parse errors with line number and message. Header validation for BudgeTrak CSV format. Empty file detection. Partial results preserved.

14.2 Encryption/Decryption Errors

Minimum data size validation. Poly1305 authentication tag verification. Invalid password/key throws authentication failure. Per-field encryption errors in sync are logged and the record is skipped (hint for wrong encryption key on AEADBadTag).

14.3 Preference Loading Errors

Try-catch with defaults for enum parsing (BudgetPeriod). Multi-type cascade via getDoubleCompat() for numeric values.

14.4 Repository Loading Errors

File existence check before read. Empty/blank content returns emptyList(). Missing fields loaded with backward-compatible defaults. Corrupt records skipped with warning log.

14.5 ID Collision Handling

All ID generators use do-while retry loop. 16-bit ID space (0..65535) with collision avoidance.

14.6 Budget Calculation Safety

Division by zero guarded (periods count > 0). Day-of-month clamped to actual month length. Budget calculations rounded to 2 decimal places. NaN/Infinity guarded by SafeIO.safeDouble().

14.7 UI Input Validation

Required field validation in all dialogs. Decimal place limiting per currency. Date range validation. Amount sum verification for multi-category.

14.8 Sync Error Handling

Firestore operations wrapped in try-catch with 30-second timeout. Listener auto-reconnect after 5-second delay on error. Enc hash skip prevents redundant decryption. pushRecord() falls back to set() on NOT_FOUND. Period ledger uses create-if-absent to prevent overwrites. Crash logger writes to Download/crash_log.txt.

14.9 File I/O Safety

Atomic writes via temp file + rename. Per-file ReentrantLock prevents concurrent writes. Rename fallback to copy on filesystems where renameTo() fails.

14.10 Receipt Sync Safety

Upload-first flow (upload before creating ledger entry). Orphan scan with time guard. Speed-based upload assignment with stale detection. Batch recovery cap (50 receipts). Snapshot lifecycle with error states.
